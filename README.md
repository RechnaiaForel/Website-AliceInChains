# Интернет-магазин музыкальных инструментов (Django)

Учебный проект интернет-магазина на Django. Каталог музыкальных
инструментов, корзина покупателя и оформление заказа с отправкой чека на
e-mail.

## Стек

- Python, Django 6.0
- SQLite (база данных по умолчанию)
- Bootstrap 5 (вёрстка)
- openpyxl (генерация чека в формате Excel)

## Запуск проекта

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

python manage.py migrate
python manage.py createsuperuser   # при необходимости

python manage.py runserver
```

Сайт будет доступен по адресу http://127.0.0.1:8000/

## Структура приложения (AliceInChains)

### Модели

- **Category** — категория товара (название, родительская категория).
- **Manufacturer** — производитель (название, страна, описание).
- **Product** — товар (название, описание, фото, цена, остаток на складе,
  категория, производитель).
- **Cart** — корзина пользователя (один пользователь — одна корзина).
- **CartItem** — позиция в корзине (товар + количество).
- **Order** — оформленный заказ (пользователь, адрес доставки, итоговая
  сумма).
- **OrderItem** — позиция заказа (товар, количество, цена на момент покупки).

Все модели зарегистрированы в админ-панели Django (`/admin/`).

## Страницы сайта

| Страница | URL | Описание |
| --- | --- | --- |
| Главная | `/` | Ссылки на остальные разделы сайта |
| О магазине | `/about/` | Информация о теме проекта |
| Об авторе | `/author/` | Информация об авторе лабораторной работы |
| Каталог товаров | `/catalog/` | Список товаров с фильтрами по категории и производителю и поиском по названию/описанию |
| Карточка товара | `/catalog/<id>/` | Подробная информация о товаре, добавление в корзину |
| Корзина | `/cart/` | Просмотр содержимого корзины, изменение количества, удаление позиций |
| Оформление заказа | `/checkout/` | Ввод адреса доставки и подтверждение заказа |
| Вход / выход | `/accounts/login/`, `/accounts/logout/` | Аутентификация пользователя |

Действия с корзиной и оформление заказа доступны только авторизованным
пользователям.

## Оформление заказа

При подтверждении заказа:

1. Создаётся объект `Order` и связанные `OrderItem` на основе содержимого
   корзины.
2. Остаток товара (`quantity_in_stock`) уменьшается на купленное количество.
3. Формируется чек в формате Excel (`.xlsx`) с перечнем товаров и итоговой
   суммой.
4. Чек отправляется на e-mail пользователя через `send_mail`/`EmailMessage`.
5. Корзина пользователя очищается.

## Настройка отправки email

По умолчанию письма не уходят на реальный почтовый сервер, а выводятся в
консоль (`EMAIL_BACKEND = 'django.core.mail.backends.console.EmailBackend'`)
— это удобно для проверки без настройки SMTP.

Для отправки настоящих писем в `nutshell/settings.py` нужно заменить backend
на SMTP и указать параметры почтового сервера (адрес, логин, пароль
приложения) — соответствующий пример закомментирован в файле настроек.
